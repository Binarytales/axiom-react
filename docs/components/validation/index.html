Validation

The validation components are here to provide a consistent approach to form validation. They work using React's render callback pattern (functions as children), that are provided with information of singular input validity and the overal validity of the form.

The pattern set out is to provide clear and not overwheling feedback to the user around invalid inputs. To acheive this, feedback of the overall validity of a form is performed on submit, and individual validation is run inline on the first invalid input only.

Validation example
const handleClick = (validate) => {
  const validationError = validate();

  this.setState({
    validationError
  });

  if (!validationError) {
    console.log('Submit!');
  }
};
 const { validationError } = this.state;

<Validation requiredError="All fields must be filled in!">
  { (validate) => (
  <Validate required value="somevalue">
    { (isValid) => isValid === true }
  </Validate>

  <Validate error={ ()=> 'Needs a capital letter!' } patterns={ [/^.*[A-Z].*$/] } required value="somevalue"> { (isValid) => isValid === false }
  </Validate>

  { validationError }

  <Button onClick={ ()=> handleClick(validate) }> Validate
  </Button>
  ) }
</Validation>

Show <Base> properties
children
Requiredfunc

Function that returns React.node type and is provided with information of the validity of the inputs state. @param {bool} isValid Inline validity state for that input @param {bool} hasMetRequired Returns the current required validity @param {function} checkPatternMet Takes a pattern and returns the validity of that pattern

error
func

Function that given the inputs invalidations, returns an error message which is given by the top level Validator component.

patterns
arrayOf

Array of patterns to be run against this input.

required
bool

If this is a required field

value
any

Current value to run against the validations
